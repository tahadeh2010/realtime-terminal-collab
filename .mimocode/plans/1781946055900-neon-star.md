# TASK-008 - Create ConnectionManager

## Design

### 1. Connection Data Structure

```go
type Connection struct {
    ID   string
    Conn *websocket.Conn
}
```

Minimal struct holding client ID and raw WebSocket connection. No session awareness — that's a separate concern.

### 2. Register / Unregister Flow

```
Register(id, conn)   → add to map[id]*Connection
Unregister(id)       → remove from map
GetConnection(id)    → retrieve from map
```

All operations protected by `sync.RWMutex` for thread safety.

### 3. SessionManager vs ConnectionManager

| Concern | SessionManager | ConnectionManager |
|---------|---------------|-------------------|
| Layer | Application | Transport |
| Manages | Session lifecycle | Active WebSocket connections |
| Storage | SessionStore (持久化) | In-memory map |
| Knows about | Sessions, Clients | Raw connections only |
| Dependencies | SessionStore interface | None (transport-only) |

### 4. What MUST NOT Be in ConnectionManager

- Session creation/deletion
- WebSocket upgrade logic
- Message broadcasting
- PTY logic
- Role/permission checks

## Implementation

### File: `internal/transport/connection_manager.go`

```go
package transport

import (
    "sync"
    "github.com/gorilla/websocket"
)

type Connection struct {
    ID   string
    Conn *websocket.Conn
}

type ConnectionManager struct {
    mu          sync.RWMutex
    connections map[string]*Connection
}

func NewConnectionManager() *ConnectionManager
func (cm *ConnectionManager) Register(id string, conn *websocket.Conn)
func (cm *ConnectionManager) Unregister(id string)
func (cm *ConnectionManager) GetConnection(id string) (*Connection, bool)
```

### Why `*websocket.Conn`?
Type-safe, direct access. The manager lives in the transport layer, so importing gorilla/websocket is acceptable.
